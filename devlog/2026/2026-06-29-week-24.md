## Development Notes

The week opened by continuing the Dynamixel work that had just landed. A small CLI cheatsheet update was followed by kernel work on lag compensation and timing resynchronization, tightening how the runtime recovers timing alignment under load. The first large robotics commit then reshaped the new Dynamixel controller around clearer head and body robot definitions, updated servo parameter files, and renamed smoke-test models to match the reorganized robot layouts.

That controller work did not stay static through the week. A second large correction pass reworked the module implementation again, updated servo-chain and configuration helpers, revised the Epi head and body JSON files, and simplified several live smoke tests. The accompanying `Epi.ikg` update kept the project model aligned with those controller changes, so the robotics path ended the week in a more coherent state than it had at the start.

The browser-side and test work broadened the runtime surface around asynchronous execution. `WebUIWidgetGrid.js` gained a focused update, and a much larger cleanup pass added async lifecycle fixtures, queued-control and open/stop/pause tests for both Python and C++ async modules, and new handling for hiding internal test assets from the WebUI. That same sweep also trimmed old PythonModule surface area and reorganized kernel test resources, making the async path more explicitly exercised in the repository.

Documentation then caught up with the code. The API reference, async-modules guide, PythonModule readme, and both the `.ikc` and `.ikg` references were expanded, and the final commit added more error-handling notes. By the end of the week, the repository had a better documented and more thoroughly exercised async runtime, together with a more realistic Dynamixel controller configuration for Epi.
