## Summary

This week adjusted browser polling during fast-forward execution. Birger increased the JavaScript run timer so that, in run mode, the browser fetched data every 100 ms rather than polling more aggressively.

## Highlights

- Increased the WebUI JavaScript run timer.
- Reduced browser data-fetch frequency during fast-forward run mode.

## Narrative

The week is a focused WebUI performance adjustment. In fast-forward run mode, the browser no longer needs to fetch data at the same rate as step-by-step or realtime interaction. Birger's timer change reduces update pressure during continuous execution and makes the browser-side interface less demanding while the simulation runs quickly.

## Major Commits And Updates

- `46842ab` - Increased the JavaScript run timer for fast-forward mode.

## Project State At The End Of The Week

- Browser polling during run mode was throttled to a 100 ms data-fetch interval.
- The WebUI had a better balance between responsiveness and runtime load.
