## Summary

This week brought a new HTML WebUI direction together with a new FFMpeg-based video input module. Work recorded under `ikaros-project` reorganized WebUI assets, fixed controls, added canvas objects, rewrote line rendering, added image loading controls, and introduced profiling. Birger contributed the `VideoInputFile` module, video-stream fixes, and code cleanup. Christian Balkenius added image-object load control.

## Highlights

- Added `VideoInputFile`, based on FFMpeg.
- Introduced a new HTML WebUI and reorganized paths for icons and stylesheets.
- Fixed WebUI controls and removed obsolete viewer assets.
- Added color BMP image support.
- Reworked canvas objects and line rendering.
- Added the first canvas WebUI object, `CLines`.
- Added image-object load control and profiling.
- Completed and merged the video input module work with the HTML WebUI branch.
- Added initial path-object support and inspector path fixes.

## Narrative

The week is a major step in making Ikaros more usable through the browser. The new HTML WebUI work reorganized the interface files, corrected asset paths, fixed controls, and began moving visualization objects onto canvas. Line rendering was rewritten and `CLines` became the first canvas WebUI object, establishing a direction for richer browser-side views.

In parallel, Birger added `VideoInputFile` on top of FFMpeg. The module work included fixes to consistently retrieve frames from the video stream and final cleanup before the branch was merged with the HTML WebUI work. This connected the interface work with a practical media input path.

Image handling also advanced. Color BMP support landed, image objects gained load control, and profiling was introduced. By the end of the week, the WebUI had a stronger HTML/canvas foundation and the repository had a new file-based video input module.
